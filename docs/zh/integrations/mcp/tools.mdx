## 概览 {#overview}

Browserbase MCP 服务器提供了一套完整的浏览器自动化与会话管理工具。借助这些工具，你可以执行页面导航、截取屏幕、管理 Cookie，并可同时管理多个浏览器会话。

## 核心浏览器自动化工具 {#core-browser-automation-tools}

以下是使用自然语言命令进行现代网页自动化的主要工具。

<Accordion title="browserbase_stagehand_navigate">
  在浏览器中打开任意 URL

  <ParamField path="url" type="string" required>
    要访问的 URL
  </ParamField>
</Accordion>

<Accordion title="browserbase_stagehand_act">
  使用自然语言在网页上执行操作

  <ParamField path="action" type="string" required>
    要执行的操作（例如：“点击登录按钮”、“填写表单字段”）
  </ParamField>
</Accordion>

<Accordion title="browserbase_stagehand_extract">
  从当前页面提取所有文本内容（会过滤 CSS 和 JavaScript）

  <Info>无需输入参数</Info>
</Accordion>

<Accordion title="browserbase_stagehand_observe">
  观察并发现网页上的可操作元素

  <ParamField path="instruction" type="string" required>
    观察的具体指令（例如：“找到登录按钮”、“定位搜索表单”）
  </ParamField>
</Accordion>

<Accordion title="browserbase_screenshot">
  捕获当前页面的 PNG 截图

  <Info>无需输入参数</Info>

  <ResponseField name="image" type="string">
    Base64 编码的 PNG 数据
  </ResponseField>
</Accordion>

<Accordion title="browserbase_stagehand_get_url">
  获取当前浏览器页面的 URL

  <Info>无需输入参数</Info>

  <ResponseField name="url" type="string">
    完整的 URL，包括协议、域名、路径，以及任何查询参数或片段
  </ResponseField>
</Accordion>

<Accordion title="browserbase_stagehand_get_all_urls">
  获取所有活动浏览器会话的当前 URL

  <Info>无需输入参数</Info>

  <ResponseField name="sessionUrls" type="object">
    会话 ID 到其当前 URL 的映射（JSON 格式）
  </ResponseField>
</Accordion>

## 单会话管理 {#single-session-management}

传统的一体化活动浏览器会话方案。适用于基础自动化任务，更为简洁，并自动管理当前活动会话。

<Accordion title="browserbase_session_create">
  使用 Browserbase 创建或复用云端浏览器会话，并附带已完全初始化的 Stagehand

  <ParamField path="sessionId" type="string">
    可选的会话 ID（用于使用/复用）。未提供则创建新会话
  </ParamField>
</Accordion>

<Accordion title="browserbase_session_close">
  关闭当前 Browserbase 会话，断开浏览器连接，并清理 Stagehand 实例

  <Info>无需输入参数</Info>
</Accordion>

## 多会话管理 {#multi-session-management}

适用于复杂自动化工作流的高级方案，可并行运行多个浏览器会话。每个会话都维护独立的状态、cookie 和浏览器上下文。

### 会话生命周期管理 {#session-lifecycle-management}

<Accordion title="multi_browserbase_stagehand_session_create">
  创建一个新的独立 Stagehand 浏览器会话，具备完整的网页自动化能力

  <ParamField path="name" type="string">
    便于追踪的人类可读名称（例如：'login-flow'、'data-scraping'）
  </ParamField>
</Accordion>

<Accordion title="multi_browserbase_stagehand_session_list">
  列出当前所有活跃的 Stagehand 浏览器会话及其详细元数据

  <Info>无需输入参数</Info>
</Accordion>

<Accordion title="multi_browserbase_stagehand_session_close">
  关闭并清理指定的 Stagehand 浏览器会话

  <ParamField path="sessionId" type="string" required>
    要关闭的精确会话 ID（操作不可撤销）
  </ParamField>
</Accordion>

### 会话级自动化工具 {#session-specific-automation-tools}

所有核心浏览器自动化工具均提供会话级变体：

<Accordion title="multi_browserbase_stagehand_navigate_session">
  在特定浏览器会话中导航至指定 URL

  <ParamField path="sessionId" type="string" required>
    要使用的会话 ID
  </ParamField>

  <ParamField path="url" type="string" required>
    目标 URL
  </ParamField>
</Accordion>

<Accordion title="multi_browserbase_stagehand_act_session">
  在特定浏览器会话中使用自然语言执行动作

  <ParamField path="sessionId" type="string" required>
    要使用的会话 ID
  </ParamField>

  <ParamField path="action" type="string" required>
    要执行的动作
  </ParamField>
</Accordion>

<Accordion title="multi_browserbase_stagehand_extract_session">
  从特定浏览器会话中提取结构化信息

  <ParamField path="sessionId" type="string" required>
    要使用的会话 ID
  </ParamField>

  <ParamField path="instruction" type="string" required>
    要从页面提取的内容
  </ParamField>
</Accordion>

<Accordion title="multi_browserbase_stagehand_observe_session">
  在特定浏览器会话中观察并查找可操作元素

  <ParamField path="sessionId" type="string" required>
    要使用的会话 ID
  </ParamField>

  <ParamField path="instruction" type="string" required>
    观察目标（例如："find the login button"）
  </ParamField>

  <ParamField path="returnAction" type="boolean">
    是否返回可执行的动作
  </ParamField>
</Accordion>

<Accordion title="multi_browserbase_stagehand_get_url_session">
  获取特定浏览器会话的当前 URL

  <ParamField path="sessionId" type="string" required>
    要使用的会话 ID
  </ParamField>

  <ResponseField name="url" type="string">
    完整 URL，包括协议、域名、路径以及任意查询参数或片段
  </ResponseField>
</Accordion>
