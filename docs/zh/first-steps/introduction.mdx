Stagehand 是一款浏览器自动化框架，可通过自然语言与代码控制网页浏览器。借助将 AI 的能力与代码的精确性相结合，Stagehand 让网页自动化更加灵活、易于维护，并且真正可靠。

## 浏览器自动化的痛点 {#the-problem-with-browser-automation}

像 Playwright 和 Puppeteer 这样的传统框架迫使你编写易碎的脚本，任何 UI 变动都可能使其失效。Web 代理声称可用 AI 解决这一问题，但却让你陷入行为不可预测的困境。

**你被夹在两个糟糕的选择之间：**

- **过于脆弱**：传统选择器在网站变化时会失效
- **过度“代理化”**：AI 代理行为不可预测且难以调试

## 走近 Stagehand {#enter-stagehand}

Stagehand 通过四个强大的原语，为你带来“两全其美”的组合，让你精确掌控需要使用的 AI 程度：

<CodeGroup>
  ```typescript TypeScript
  // Act - 使用自然语言执行操作
  await page.act("click the login button");

  // Extract - 提取结构化数据
  const { price } = await page.extract({
    schema: z.object({ price: z.number() })
  });

  // Observe - 发现可执行动作
  const actions = await page.observe("find submit buttons");

  // Agent - 自主完成整条工作流
  const agent = stagehand.agent({
      provider: "anthropic",
      model: "claude-sonnet-4-20250514",
      options: {
        apiKey: process.env.ANTHROPIC_API_KEY,
      },
  })
  await agent.execute("apply for this job");
  ```

  ```python Python
  # Act - 使用自然语言执行操作
  await page.act("click the login button")

  # Extract - 提取结构化数据
  result = await page.extract(
    schema={"price": float}
  )

  # Observe - 发现可执行动作
  actions = await page.observe("find submit buttons")

  # Agent - 自主完成整条工作流
  await agent.execute("apply for this job")
  ```
</CodeGroup>

## 开发者为何选择 Stagehand {#why-developers-choose-stagehand}

- **精确控制**：将 AI 驱动的动作与确定性代码相结合。你可以精确掌控使用多少 AI。

- **真正可复现**：准确保存并重放动作。做浏览器自动化时，不再出现“在我机器上没问题”的情况。

- **可在规模上维护**：一个脚本即可自动化多个网站。站点变化时，你的自动化会自适应。

- **可组合工具**：按需选择自动化层级，结合 act()、extract()、observe() 与 agent() 使用。
